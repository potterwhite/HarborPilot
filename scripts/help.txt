USAGE:
  ./harbor [COMMAND] [OPTIONS]

COMMANDS — Image:
  image-build             Build & push Docker image to registry
  image-list              List images built by this project

COMMANDS — Container:
  container-start         Start container on this machine
  container-stop          Stop running container
  container-status        Show container status
  container-remove        Remove container

COMMANDS — Handover:
  handover-platform       Package client handover tarball for a platform
                          Usage: ./harbor handover-platform <platform.env>

OPTIONS:
  --host=<name>           Select host config by name (e.g. office-server-01)
                          Looks for configs/3_hosts/<name>.env
  -h, --help              Show this help message

ENVIRONMENT:
  V=1                     Enable debug mode (verbose output)

EXAMPLES:
  ./harbor                                    Launch interactive menu
  ./harbor image-build --host=jetson-orin-01  Build & push image for a host
  ./harbor container-start --host=jetson-orin-01  Start container locally
  ./harbor container-status --host=jetson-orin-01 Check container status
  ./harbor image-list                         List project images
  ./harbor handover-platform jetson-orin-nx-16g_ubuntu-22.04  Package handover tarball

CONFIG LAYERS (last wins):
  Layer 1 — configs/1_defaults/*.env          Global defaults
  Layer 2 — configs/2_platforms/<name>.env    Per-platform (Jetson board)
  Layer 3 — configs/3_hosts/<name>.env        Per-host (office, home, etc.)
